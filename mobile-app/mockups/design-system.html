SetaLink is a VPN used in low-light, high-stress contexts. Readability is non-negotiable, even with glow effects active. Every glow is decorative — text contrast must pass without it.

Glow Limits for Readability
DOTest text contrast with glows disabled. Glow is decorative only — text must pass WCAG AA (4.5:1 for body, 3:1 for large text) before glow is applied.
DOCap glow box-shadow opacity at 0.28 for borders and 0.24 for spread glow. Beyond this, the glow begins to bleed onto adjacent text and reduces readability.
NONever place body text adjacent to an active glow without a dark background buffer. The glow halo must not reach readable text elements.
NONever drop text size below 10px in production. Micro labels at 10px are the hard floor. Navigation labels minimum 9px only for the inactive 9px spec — active labels must be 10px+.
NONever use muted text (#3D5570) for actionable elements, error states, or any text the user must act on. Reserve muted only for purely decorative or supplementary labels.

08
Brand Personality

SetaLink is not a utility. It is a premium intelligence layer between the user and the internet. The interface should feel like a calm, confident co-pilot — not a technical dashboard.

Six Core Feelings
Calm
Slow animations, generous spacing, restrained glow. Nothing on screen is fighting for attention. The user feels in control, not overwhelmed.
Intelligent
Protocol data in monospace, AI decisions surfaced in real time, adaptive routing. The app communicates what it's doing without requiring the user to understand it.
Stealth
Dark backgrounds, subtle borders, glow only where necessary. The UI doesn't shout. It whispers capability. Like a matte-black instrument.
Cinematic
Staggered section reveals, breathing orbs, orbit rings. Every state transition tells a small story. Opening the app should feel like a brief scene.
Premium
High contrast, precise typography, perfect radius consistency. Reference: Apple system UI, Arc Browser, Linear, Notion Calendar. The details are right.
Trustworthy
Technical data is shown clearly. Health checks are visible. The user always knows the protocol, latency, and connection state. Transparency is the trust mechanism.
Visual Voice — Do & Don't
DOReserve emerald for exactly three contexts: connection state (the connect button + status dot), active/selected items (nav, mode cards), and primary CTAs. Everywhere else: neutral.
DOMake the AI center button the emotional focal point. It should feel slightly elevated, slightly alive, and always glowing softly — even when the screen is idle.
DOLet the dark void breathe. Extra whitespace (dark-space) between cards is not waste — it is the premium signal. Tight layouts feel cheap.
DOReference Linear, Arc, Nothing OS, Tailscale, and Mullvad for visual compass. These products share SetaLink's commitment to restraint + capability.
NONever stack multiple glow layers. One soft glow layer is intentional. Two layers is uncertain. Three layers is panic.
NONever animate more than three elements simultaneously. The pulse rings, the breathing orb, and the status dot — that is the maximum active animation state on any screen.
NONever use color purely decoratively. Every color appearance must carry semantic meaning: emerald = active/safe, amber = warning, red = danger, blue = info/data flow.
NONever use gradients on text. The only text gradient use is the "SetaLink" name on the splash screen wordmark. All other text is flat.
